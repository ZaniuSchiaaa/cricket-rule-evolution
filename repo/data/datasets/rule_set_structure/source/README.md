Different time periods of rule sets were enumerated with different enumeration systems. Thus, different regex patterns were required to automatically detect the layers of depth in these rule sets. In this repository, we are including the code file for one such sample time period: 2017-2019.  

While the code is fairly accurate at recreating the tree structure of the rule sets in the .yaml files, there are still slight discrepancies. When inspecting the trees, we identified and rectified some of these discrepancies by hand. Hence, the .yaml files that this code file generate may not fully match the .yaml files in the data folder. The .yaml files generated by this code file has the suffix "-raw" for that reason. For the most accurate .yaml files, refer to the files in the data folder. 

The code contains the flag IS_FLATTENED, granting the option to generate the flattened tree structure. Note that while both `original` and `flattened` .yaml files are in the repository, we only used the tree structure in the `flattened` .yaml files for any analysis regarding the rule set structure.
